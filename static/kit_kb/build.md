# Blueprint Builds (Build) — vibe-code apps

Describe an app or tool in plain words and it builds a working single-file web app
in a live preview. Deep how-to below.

## What Blueprint Builds is
A "vibe-coding" room: you type what you want in plain language, hit Build, and a
working single-file web app appears in the live preview. No coding required — you
describe, it builds.

## Building something
Type what you want in the box (e.g. "a tip calculator with dark mode") and hit
Build. A working app shows up in the preview. Be specific about what it should do
and look like for a closer first result.

## Iterating (keep talking)
Keep refining by talking to it: "make the button bigger", "add dark mode", "store
entries in local storage". Each message stacks changes onto the current app. It
also auto-fixes its own runtime errors as it builds, so you can keep steering
instead of debugging.

## Reference images / video
You can attach images or video as reference for the look or behavior you want, so
the build matches a design you have in mind.

## Previewing on devices
Preview the app in phone, tablet, or desktop frames to check how it looks at
different sizes before you finish.

## Saving builds
Finished builds save to your library so you can reopen, continue, or revisit them
later.
